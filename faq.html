Where can I get help?

Feel free to ask questions in our Gitter channel.

Where can I report a bug?

Please file an issue on GitHub.

Who maintains and develops Zing?

Zing is built by the localization team at Evernote.

Is Zing actively developed?

While the current feature set already fulfills the needs of Evernote, Zing is actively supported by fixing bugs and upgrading to newer versions of libraries.

Is Zing production-ready?

Yes. The Evernote Translation Server runs Zing, and it's an important part of the day-to-day business.

Is Zing stable across versions?

Zing adheres to Semantic Versioning, and because it hasn't reached a 1.0 version yet, anything might change anytime and there can be backwards incompatible changes between releases.

Why does Zing only work with PO files?

The guiding principle for Zing is integrating well with Serge in order to provide a flexible continuous localization workflow. As such, it only requires to support an interchange file format, and Gettext PO has resulted to be a perfect fit in our experience of over 8 years.
